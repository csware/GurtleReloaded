Setting up Gurtle for Google Code Hosted Projects

For latest information, see online documentation.

The purpose of Gurtle is to allow you to easily access the list of issues in your projects hosted at Google Code when performing a commit using TortoiseSVN. From within the Commit dialog box, you will be able to select one or more issues listed online. The number and summary of each selected issue will then be appended to your commit log message, serving as a cross-reference for the future.

In order to enable access to issues for a project during commits, you need to configure TortoiseSVN and Gurtle together. The steps for this are already described in the TortoiseSVN documentation, under “Issue Tracker Integration.”

When you see the Issue Tracker Integration/Add dialog box appear in TortoiseSVN, you will need to supply three pieces of information. In the Working Copy Path field, enter the absolute path to the working copy of your project. From the Provider list, select Gurtle (Google Code). Finally, the Parameters field should contain the following string, where PROJECT should be replaced with the name of your Google Code project:

project=PROJECT

For example, for this project, Gurtle, which is located at http://code.google.com/p/gurtle/, the Parameters field would read project=gurtle. The project name is case-sensitive so you should enter it correctly, usually all in lower case letters.

That is it. The next you are ready to commit a change pertaining to an issue, you should see a button with the text Select Issue appear in the Commit dialog box from TortoiseSVN.

Alternatively, you can configure Gurtle with the Subversion properties bugtraq:providerparams and bugtraq:provideruuid. Set the value of the bugtraq:provideruuid property to {91974081-2DC7-4FB1-B3BE-0DE1C8D6CE4E}, which is the COM UUID of Gurtle. Set the value of the bugtraq:providerparams to the value project=my-google-project-name, with my-google-project-name being the name of your Google Code project. For example, for Gurtle, this would become project=gurtle.
